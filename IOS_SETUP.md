# HealingWings iOS App — Setup Guide

This branch (`ios-app`) wraps the existing React/Vite web app in a native iOS shell
using [Capacitor](https://capacitorjs.com). The web app code is **unchanged** — Capacitor
just embeds the built `dist/` bundle into a WKWebView inside a native Xcode project.

> **This branch is never merged to `main`.** It exists solely for iOS testing/distribution.
> If `main` gets new features, run `git merge main` into this branch to pull them in.

---

## Prerequisites (Mac only)

| Tool | Version | Install |
|------|---------|---------|
| macOS | 13+ (Ventura or later) | — |
| Xcode | 15+ | Mac App Store |
| Node.js | 18+ | [nodejs.org](https://nodejs.org) |
| CocoaPods | 1.14+ | `sudo gem install cocoapods` |

---

## First-Time Setup (run once)

```bash
# 1. Switch to this branch and go into the frontend folder
git checkout ios-app
cd frontend

# 2. Install all dependencies (including Capacitor packages)
npm install

# 3. Build the React app
npm run build

# 4. Generate the Xcode project (creates frontend/ios/ folder)
npx cap add ios

# 5. Copy the web build into the native shell + install CocoaPods
npx cap sync ios

# 6. Open in Xcode
npx cap open ios
```

That's it. Xcode will open with the project at `frontend/ios/App/App.xcworkspace`.

## Running on a Simulator

In Xcode:
1. Select a simulator from the device dropdown (e.g. "iPhone 15 Pro")
2. Press **⌘R** (Run)

The app loads, connects to the Azure backend (`intex-group4-5-backend-dna7b3ddfvf8dxgb.centralus-01.azurewebsites.net`), and behaves identically to the web version.

---

## Running on a Physical iPhone

1. Plug in your iPhone via USB
2. In Xcode → Signing & Capabilities tab → select your Apple Developer Team
3. Set the Bundle Identifier to `org.healingwings.app` (or any unique reverse-DNS ID your team owns)
4. Select your physical device from the dropdown → ⌘R

---

## Distributing via TestFlight

1. In Xcode → Product → Archive
2. In the Organizer window → Distribute App → App Store Connect → Upload
3. In [App Store Connect](https://appstoreconnect.apple.com) → TestFlight → add testers

> Requires a paid Apple Developer account ($99/year).

## What Capacitor Does / Doesn't Change

| | Details |
|---|---|
| **Web app code** | Untouched — same React components, same routes, same auth |
| **API calls** | All go to Azure backend over HTTPS — same as the web app |
| **Auth cookies** | Capacitor uses WKWebView which supports cookies; login/logout/MFA all work |
| **Google OAuth** | Requires adding a custom URL scheme in Xcode (see below) |
| **Deep links** | Not configured — add `@capacitor/app` listener if needed |

### Google OAuth on iOS (extra step)

WKWebView can't complete the Google OAuth redirect without a registered URL scheme.
In Xcode → Info tab → URL Types → add a new entry:

- **Identifier:** `com.google.oauth`
- **URL Schemes:** your Google OAuth client ID reversed (e.g. `com.googleusercontent.apps.XXXX`)

This is only needed if you want Google sign-in to work in the native shell.
Email/password login works out of the box.
